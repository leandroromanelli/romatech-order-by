# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
| < Latest | No       |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **DO NOT** open a public GitHub issue
2. Email: security@romatech.dev
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- Acknowledgment: within 48 hours
- Initial assessment: within 1 week
- Fix release: depends on severity (critical: 72h, high: 1 week, medium: 2 weeks)

## Scope

This policy applies to all packages in the @romatech npm scope and the Romatech.Extensions.* NuGet packages.

## Recognition

We appreciate responsible disclosure and will credit reporters in release notes (unless anonymity is preferred).
